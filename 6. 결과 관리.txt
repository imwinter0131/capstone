6. 결과 관리

학습이 끝난 작업의 성능 지표, epoch 변화, 결과 파일 경로, 로그, 실험 비교를 확인하는 기능입니다.

5. 학습 관리에서 생성하고 실행한 학습 작업을 기준으로 결과를 보여줍니다.
완료된 학습뿐만 아니라 실패하거나 중지된 학습도 확인할 수 있어야 원인 분석이 가능합니다.

결과 관리는 별도의 results 테이블을 새로 만들지 않습니다.
학습 작업을 저장하는 training_jobs 테이블의 result_json, logs_json, run_dir 값을 사용합니다.

실행 프로세스

1. 결과 확인 위치

좌측 메뉴에서 Training 버튼 선택

/projects/{project_id}/training 주소 이동

App.jsx에서 TrainingManage.jsx 표시

TrainingManage.jsx 오른쪽 Live Result 패널에서 학습 결과 확인


2. 결과 목록 로딩

TrainingManage.jsx에서 다음 API 호출

GET /projects/{project_id}/training-jobs?user_id={user_id}

백엔드는 프로젝트 소유자 확인 후 training_jobs 테이블에서 결과 대상 학습 작업을 조회합니다.

기본 조회 대상:

* COMPLETED
* FAILED
* STOPPED

목록에서 확인 가능한 항목:

* 학습 이름
* 작업 유형
* 상태
* YOLO 모델
* 데이터셋 이름
* 주요 성능 지표
* 완료 시각
* 실행 시간


3. 선택한 학습 결과 표시

학습 관리 화면에서 학습 카드를 선택하면 오른쪽 Live Result 패널의 내용이 바뀝니다.

탐지 Detect 주요 지표:

* mAP50-95
* mAP50
* Precision
* Recall
* F1

분류 Classify 주요 지표:

* Top-1 Accuracy
* Top-5 Accuracy
* F1

학습이 실행 중이면 진행률, 현재 Epoch, 최신 로그를 함께 표시합니다.


4. 결과 상세 조회

결과 카드를 선택하면 이미 불러온 training_jobs 응답에서 해당 학습의 result_json과 logs_json 값을 표시합니다.

필요하면 다음 API로 단일 학습 로그를 다시 조회할 수 있습니다.

GET /projects/{project_id}/training-jobs/{job_id}/logs?user_id={user_id}

상세 화면에서 확인 가능한 항목:

* 학습 이름
* 상태
* 데이터셋
* 전처리 preset
* YOLO 모델
* Optimizer
* Image Size
* Epochs
* 시작 시각
* 완료 시각
* 실행 시간
* 결과 폴더
* 주요 metric
* epoch별 metric
* artifact 경로
* 학습 로그


5. epoch 그래프 표시

학습 결과에 epoch_metrics 값이 있으면 그래프로 표시합니다.

표시 지표:

* Train Loss
* Validation Loss
* 주요 성능 지표

ultralytics 학습으로 results.csv가 생성된 경우 백엔드가 results.csv를 읽어 epoch별 지표로 변환합니다.

개발 환경에서 ultralytics 없이 시뮬레이션 학습을 실행한 경우에도 확인용 epoch 지표를 생성합니다.


6. 결과 파일 표시

백엔드는 run_dir 기준으로 대표 결과 파일의 예상 경로를 구성합니다.

파일이 존재하는 경우에는 exists 값을 true로 내려줍니다.
파일이 아직 없거나 시뮬레이션 학습인 경우에는 exists 값이 false일 수 있습니다.

화면에서는 파일 이름과 경로를 함께 표시합니다.


7. 실험 비교

결과 비교는 별도 화면으로 만들지 않습니다.

현재 구현에서는 학습 관리 화면에서 학습 작업을 선택해 하나씩 결과를 확인합니다.
오른쪽 Live Result 패널에서 완료된 학습의 상위 결과를 Top Runs 영역으로 표시합니다.

Top Runs에서 확인 가능한 항목:

* 학습 이름
* 주요 metric
* metric 이름
* 순위

Top Runs 항목을 선택하면 해당 학습 결과가 오른쪽 패널에 표시됩니다.


8. 결과 이미지 미리보기

YOLO가 생성한 대표 이미지가 있으면 Live Result 패널에서 바로 확인합니다.

미리보기 대상:

* results.png
* confusion_matrix.png
* labels.jpg
* train_batch 이미지
* val_batch 이미지

백엔드는 프로젝트 저장 경로 내부의 이미지 파일만 반환합니다.
프로젝트 폴더 밖의 파일은 열 수 없습니다.


9. 결과 기반 재학습

Retune from Result 버튼 선택

선택한 학습의 설정을 복사해 새 학습 생성 모달을 엽니다.

복사되는 항목:

* 작업 유형
* 데이터셋
* 전처리 preset
* YOLO 모델
* Optimizer
* Image Size
* Epochs
* Batch Min / Batch Max
* LR Initial Min / LR Initial Max
* Momentum Min / Momentum Max

사용자는 복사된 값을 직접 수정한 뒤 새 학습으로 저장합니다.

기존 학습을 덮어쓰지 않습니다.
항상 새 학습 작업으로 생성합니다.


10. 튜닝 제안

Live Result 패널에서 현재 선택한 학습 결과를 기준으로 다음 실험 방향을 제안합니다.

예시:

* Validation Loss가 높으면 LR Initial Max 축소 제안
* Recall이 낮으면 Image Size 또는 Epochs 증가 제안
* Precision이 낮으면 라벨 품질 확인과 LR 범위 축소 제안
* 실패한 학습이면 로그와 데이터셋 구조 확인 제안

튜닝 제안은 자동으로 학습 값을 변경하지 않습니다.
사용자가 결과를 확인하고 값을 직접 수정해 재학습합니다.


11. 실패 결과 분석

FAILED 상태의 학습도 결과 목록에 표시합니다.

확인 가능한 항목:

* 실패 시점
* 오류 메시지
* 마지막 로그
* 사용한 데이터셋
* 사용한 전처리 preset
* 사용한 YOLO 모델

데이터셋 구조 오류, data.yaml 누락, 모델 파일 오류 같은 문제를 추적할 수 있습니다.

주의 사항

학습 결과는 5. 학습 관리에서 학습을 실행한 뒤 생성됩니다.
학습을 한 번도 실행하지 않으면 Training 화면의 결과 패널에는 표시할 결과가 없습니다.

실제 YOLO 학습 결과는 ultralytics가 생성하는 results.csv와 모델 파일을 기준으로 표시합니다.

ultralytics가 설치되지 않은 개발 환경에서는 시뮬레이션 학습 결과가 표시됩니다.
이 경우 metric과 epoch 그래프는 기능 확인용 값입니다.

모델 등록, 모델 버전 관리, 배포 대상 선정은 결과 관리에서 직접 처리하지 않습니다.
결과 관리에서 성능을 확인한 뒤 다음 모델 관리 단계에서 다루는 구조가 적절합니다.
