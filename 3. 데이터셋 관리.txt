3. 데이터셋 관리

프로젝트 안에서 YOLO 학습에 사용할 zip 데이터셋을 업로드하고, 데이터셋 구조 리포트를 확인하는 기능입니다.

이번 수정에서 데이터셋 관리는 순수하게 데이터셋 파일과 리포트만 담당하도록 정리했습니다.
기존에 이 화면에 있던 YOLO 모델 선택, Optimizer, Optuna Search Space, 학습 생성 기능은 5. 학습 관리로 이동했습니다.

--------------------------------------------------------------------------------------------------------------------
기능 범위

포함 기능

* 프로젝트별 데이터셋 목록 조회
* 새 데이터셋 업로드
* 데이터셋 이름 입력
* 설명 입력
* 작업 유형 선택: 탐지 Detect / 분류 Classify
* zip 데이터셋 파일 업로드
* 업로드 전 zip 데이터셋 분석
* 데이터셋 리포트 생성
* 데이터셋 삭제

제외 기능

* YOLO 모델 선택
* Optimizer 설정
* Image Size 학습 설정
* Optuna Search Space 설정
* 학습 작업 생성

위 제외 기능은 5. 학습 관리에서 담당합니다.
--------------------------------------------------------------------------------------------------------------------



--------------------------------------------------------------------------------------------------------------------
파일별 역할

Backend 폴더 (FastAPI + PostgreSQL)

database.py: PostgreSQL 데이터베이스 연결 설정

models.py: User, Project, Dataset, TrainingJob, PreprocessingPipeline 테이블 정의

main.py: 로그인/회원가입 API + 프로젝트 관리 API + 데이터셋 관리 API + 전처리 관리 API + 학습 관리 API

venv/: 백엔드 전용 가상환경 폴더


Frontend 폴더 (React)

src/DatasetManage.jsx: 프로젝트 내부 데이터셋 목록, 업로드, 리포트, 삭제 화면

src/DatasetManage.css: 데이터셋 관리 화면 전용 다크 UI 디자인

src/App.jsx: /projects/{project_id} 주소를 DatasetManage.jsx와 연결
--------------------------------------------------------------------------------------------------------------------



--------------------------------------------------------------------------------------------------------------------
실행 프로세스

1. 프로젝트 내부 이동

프로젝트 목록에서 Open 버튼 선택

/projects/{project_id} 주소 이동

App.jsx에서 DatasetManage.jsx 표시


2. 데이터셋 목록 조회

DatasetManage.jsx에서 현재 로그인 사용자 user_id와 project_id 사용

GET /projects/{project_id}/datasets?user_id={user_id}

FastAPI에서 현재 사용자 소유 프로젝트 확인

PostgreSQL datasets 테이블에서 프로젝트에 속한 데이터셋 목록 조회


3. 데이터셋 업로드

New Dataset 버튼 선택

입력 항목:

* 데이터셋 이름
* 설명
* 작업 유형: Detect / Classify
* zip 데이터셋 파일

분석 버튼 선택

POST /datasets/analyze-zip

zip 파일을 임시로 백엔드에 전송해서 내부 구조를 분석합니다.

분석 결과는 업로드 전에 화면 아래 리포트 영역에 바로 표시합니다.

POST /projects/{project_id}/datasets

multipart/form-data 형식으로 user_id, dataset_name, description, task_type, dataset_zip 전송


4. 데이터셋 저장

FastAPI에서 프로젝트 소유자 확인

데이터셋 이름 중복 확인

zip 확장자 확인

프로젝트 저장 경로 아래 datasets 폴더에 zip 파일 저장

PostgreSQL datasets 테이블에 데이터셋 메타데이터 저장


5. 데이터셋 리포트 생성

zip 파일 내부 구조 분석

리포트 항목:

* 전체 파일 수
* 이미지 수
* 라벨 txt 수
* train 이미지 수
* val 이미지 수
* test 이미지 수
* data.yaml 탐지 여부
* 클래스 이름
* 경고 메시지


6. 데이터셋 삭제

Delete 버튼 선택

DELETE /projects/{project_id}/datasets/{dataset_id}?user_id={user_id}

해당 데이터셋을 사용하는 학습 작업이 있으면 삭제 차단

사용 중이 아니면 DB 행과 저장된 zip 폴더 삭제

GET /projects/{project_id}/datasets

프로젝트의 데이터셋 목록 조회

업로드 전 zip 데이터셋 분석

요청 형식: multipart/form-data

필드:

* dataset_zip

새 데이터셋 업로드

요청 형식: multipart/form-data

필드:

* user_id
* dataset_name
* description
* task_type
* dataset_zip


DELETE /projects/{project_id}/datasets/{dataset_id}

데이터셋 삭제

학습에서 사용 중인 데이터셋은 삭제 불가
